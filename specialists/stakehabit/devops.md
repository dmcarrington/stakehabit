# DevOps — StakeHabit

## CI/CD Pipeline

Pipeline runs on **GitHub Actions** (`.github/workflows/ci.yml`).

### Workflow Triggers
- Push to `main`
- Pull request to `main`

### Jobs (all run on `ubuntu-latest`, JDK 17 Temurin)

| Job | Gradle Task | Artifacts |
|-----|------------|-----------|
| **lint** | `ktlintCheck` | — |
| **static-analysis** | `detekt` | — |
| **unit-tests** | `test` | Test reports (7 days) |
| **debug-build** | `assembleDebug` | Debug APK (7 days) |

Debug build depends on the three quality gates passing first.

### Gradle Caching
Uses `gradle/actions/setup-gradle@v4` for dependency caching. Cache is writeable on `main`, read-only on PRs.

## Static Analysis

### ktlint (v14.1.0 Gradle plugin, ktlint core v1.5.0)
- Config: `.editorconfig` (project root)
- Code style: `android_studio`
- Reporters: CHECKSTYLE + SARIF
- Kotlin version: 2.0.21

### detekt (v1.23.8)
- Config: `detekt-config.yml` (project root)
- Builds on detekt default config (`buildUponDefaultConfig = true`)
- Baseline: `app/detekt-baseline.xml` (generated by `./gradlew detektBaseline`)
- Formatting rules disabled (ktlint handles that)

### Rule Suppression
- **ktlint**: `// ktlint-disable <rule-id>`
- **detekt**: `@Suppress("detekt.RuleName")` or baseline file

## Useful Gradle Commands

```bash
# Run ktlint check
./gradlew ktlintCheck

# Auto-format with ktlint
./gradlew ktlintFormat

# Run detekt
./gradlew detekt

# Generate detekt baseline (for suppressing existing issues)
./gradlew detektBaseline

# Full local CI check
./gradlew ktlintCheck detekt test assembleDebug
```
